# Email Checker & Reputation — by Elegan4ik (Windows, offline)

## Коротко о скрипте
Скрипт берет список email из `mail.txt`, **отфильтровывает только `@yahoo.com` и `@aol.com`**, сортирует (сначала Yahoo, потом AOL) и проверяет доступность логина через Selenium (страницы Yahoo/AOL). Затем проверяет **репутацию** через Mailmeteor (web‑инструмент).  
Результаты и кэши сохраняются в файлы, чтобы не гонять одинаковые проверки повторно.

Важно: сервисы могут ограничивать автоматические запросы (fair-use / rate limiting). citeturn0search15turn0search22

## Структура файлов (что где лежит)
- `main.py` — основной скрипт.
- `mail.txt` — входной файл со списком email (может содержать любые домены).
- `license.key` — **лицензия для запуска (офлайн‑привязка к ПК)**.
- `results_YYYY-MM-DD_HH-MM-SS/` — папка результатов каждого запуска:
  - файлы доступности/ошибок (как у вас реализовано)
  - `reputation_good.txt`, `reputation_medium.txt`, `reputation_bad.txt`
  - `reputation_retry_failed.txt`, `reputation_errors.txt`
- Кэш‑файлы: кэш доступности и кэш репутации (как у вас в коде).

## Запуск через `run_main.bat` (рекомендуется)
В архиве со скриптом есть батник **`run_main.bat`** — он делает запуск максимально простым:

- переключает консоль на UTF‑8;
- переходит в папку со скриптом;
- проверяет наличие Python;
- если Python не найден — скачивает и устанавливает **Python 3.12.2 (x64)**;
- создаёт виртуальное окружение `.venv`;
- активирует окружение;
- обновляет `pip`;
- ставит зависимости (`selenium`, `webdriver-manager`, `colorama`, `requests`);
- запускает `main.py`.

### Как запускать
1) Распакуйте архив в отдельную папку (например `C:\EmailChecker\`).
2) Убедитесь, что **Google Chrome установлен** (обязательно).
3) Дважды кликните **`run_main.bat`**.

Если Windows спросит разрешение на установку/запуск — подтвердите.

### Примечания по Python 3.12
У вас используется **Python 3.12** — это нормально. Батник ставит 3.12.2, если Python отсутствует.

## Входной файл `mail.txt`
- В файле может быть **любой набор email** (gmail/msn/hotmail и т.д.).
- Скрипт сам возьмёт **только**:
  - `...@yahoo.com`
  - `...@aol.com`
- Сортировка после фильтрации:
  1) все Yahoo
  2) все AOL
- В форму логина Yahoo/AOL вводится **только часть до `@`** (например `ivan123` из `ivan123@yahoo.com`).

Пример `mail.txt`:
```
ivan123@yahoo.com
petya77@aol.com
someone@gmail.com
```

## Настройки репутации (Mailmeteor) — что менять
Блок констант в коде:
```python
REP_MAX_ATTEMPTS = ...
REP_WAIT_SECONDS = ...
REP_RETRY_BACKOFF = ...
REP_AFTER_CLICK_DELAY = ...
UNABLE_MAX_HITS = ...
```

### Рекомендованные настройки
#### Профиль: слабый ПК (медленный, мало RAM)
```python
REP_MAX_ATTEMPTS = 4
REP_WAIT_SECONDS = 180
REP_RETRY_BACKOFF = (10, 20, 40, 80)
REP_AFTER_CLICK_DELAY = 6.0
UNABLE_MAX_HITS = 3
```
И дополнительно:
- пауза между email в `check_reputation()`:
  - `time.sleep(random.uniform(6, 14))`

#### Профиль: нормальный ПК
```python
REP_MAX_ATTEMPTS = 3
REP_WAIT_SECONDS = 120
REP_RETRY_BACKOFF = (5, 10, 20)
REP_AFTER_CLICK_DELAY = 3.0
UNABLE_MAX_HITS = 2
```
Пауза между email:
- `time.sleep(random.uniform(4, 10))`

Почему это нужно: сайты могут включать rate limiting. citeturn0search22

## Частые проблемы
### “После фильтрации по yahoo/aol нечего проверять”
В `mail.txt` нет адресов `@yahoo.com` или `@aol.com` (или опечатка типа `yahooo.com`).

### Репутация часто = 0 / “Unable to check…”
Чаще всего это:
- страница не успела прогрузиться на слабом ПК
- временный сбой / ограничения по частоте  
Решение:
- профиль “слабый ПК” (выше)
- увеличить паузу между email
- не кэшировать `0`
